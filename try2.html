Recruiter Usage Report - Terms and Definitions

Last updated: 6 months ago

The LinkedIn Recruiter Usage Report includes metrics to help you measure the success and overall usage of your contract.

All reports are in Universal Coordinated Time (UTC +0). Any activity can take up to 48 hours to appear in your reports.

Heres a list of commonly used terms for Recruiter Usage Reports and their definitions.

Note: You may see blank names within your reporting due to one of the following:
  • The user has deleted their LinkedIn profile.
  • The user has enabled a privacy setting for profile visibility.
Daily active users measures the total number of active days during a time period. An active day is defined as one or more visits to Recruiter during a single day (24-hour period). This metric is an aggregated total of multiple users on desktop and mobile and is shown in the Usage Reporting graph.
A new day starts at UTC +0. If a user doesnt log out and the next day begins, an active day will be logged as long as an action is taken (for example, the user views a profile, runs a search, creates a project, etc.)
Previously, this metric was referred to as Unique Daily Logins. This metric tracked the number of visits, either single or multiple, that a particular user makes to Recruiter during a single day (24-hour period).
Active days measures the total number of active days during a time period. An active day is defined as one or more visits to Recruiter during a single day (24-hour period). This metric is based on individual usage, while Daily Active Users is an aggregate of the dashboard.
A new day starts at UTC +0. If a user doesnt log out and the next day begins, an active day will be logged if an action is taken (for example, the user views a profile, runs a search, creates a project, etc.)
Previously, this metric was referred to as Unique Daily Logins. The number of visits, either single or multiple, that a particular user makes to Recruiter during a single day (24-hour period), counts as one unique daily login.
InMails accepted measures the number InMail messages that were sent during the selected time frame and accepted within 30 days of sending the InMail message, including accepting through an email notification.This metric is an aggregated total from InMail messages accepted on desktop and mobile.
Note: InMail metrics measure all messages in Recruiter. This includes messages that dont count against the InMail balance, like messages to 1st-degree connections and open profiles.
InMails declined measures the number of InMail messages that were sent during the selected time frame and declined within 30 days.
Note: InMail metrics measure all messages in Recruiter. This includes messages that dont count against the InMail balance, like messages to 1st-degree connections and open profiles.
InMail response rate measures the percentage of InMail messages that were sent during the selected time frame and have been accepted or declined within 30 days.
Calculation: [# InMail messages accepted + # InMail messages declined] / # InMail messages sent
Note: InMail metrics measure all messages in Recruiter. This includes messages that dont count against the InMail balance, like messages to 1st-degree connections and open profiles.
InMail acceptance rate measures the percentage of InMail messages that were sent during the selected time frame and accepted within 30 days of sending the InMail, including accepting through email notification.
Calculation: [# InMail messages accepted / # InMail messages sent]
Note: InMail metrics measure all messages in Recruiter. This includes messages that dont count against the InMail balance, like messages to 1st-degree connections and open profiles.
InMail decline rate measures the percentage of InMail messages that were sent during the selected time frame and declined within 30 days of sending the InMail.
Calculation: [# InMail messages declined / # InMail messages sent]
Note: InMail metrics measure all messages in Recruiter. This includes messages that dont count against the InMail balance, like messages to 1st-degree connections and open profiles.

InMails sent includes all initial InMail messages sent during the selected time frame (using InMail and messages to 1st-degree connections and open profiles). InMail messages are counted once per thread, regardless of the number of messages exchanged. This metric is an aggregated total from messages sent on desktop and the Recruiter mobile app.

Jobs posted measures the total number of jobs successfully posted or renewed during the selected time frame (including jobs posted through Job Wrapping).
Updated stage measures the total number of candidate pipeline stage changes during the selected time frame.
New tags measures the total number of new tags added to profiles during the selected time frame.
Note: Weve updated our metrics and new tags added will be calculated starting on 4/24/2019.
Profiles saved to a project measures new profiles saved to projects in a period by a user during the selected time frame.
Profiles viewed measures the total number of profiles a user clicks into and views during the selected time frame. This metric is an aggregated total of profiles viewed on desktop and the Recruiter mobile app.
Profiles views arent distinct views. For example, if a recruiter views a profile five times in a single day, theyll have five profile views shown in reporting.
Projects created measures the total number of new projects created during the selected time frame. This metric includes projects that are created when a job is posted.
New saved search alerts measures the total number of saved search alerts enabled during the selected time frame. Each time a user toggles an alert On counts as a new saved search, including an old search that is toggled from Off to On.

Searches performed measures the number of new search actions performed during the selected time frame. This metric includes searches performed on the Recruiter mobile app or mobile browser.

New search actions include searches from the global search bar, searches performed within a project (including clearing a search and starting again) and utilizing the Find More People Like feature.

This metric was previously defined as the number of times a search was viewed by a recruiter.

New saved searches measures the total number of new searches saved during the selected time frame.

After you download the Usage report, you can sort or filter the report by the following Seat State options:

  • Active: The user currently holds a seat on your contract.
  • Inactive: The user has been permanently removed from your contract.

  • Parked: The user's seat has been temporarily deactivated from your contract.
